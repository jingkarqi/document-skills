# 文档技能对比研究

**状态：** 内部复盘草稿；对外公开前需完成许可与知识产权审查。

本仓库对比并整理两套文档处理“技能”体系（OpenAI ChatGPT Agent 工作流 vs Anthropic Claude Code 工作流），汇总相关材料、脚本与分析报告，用于研究文档自动化的工具链设计与质量保障方法。

## 目的

- 保留真实工作流的快照与分析记录
- 对比封闭式一体化工具链与开放式可组合工具链
- 为 DOCX/PDF/PPTX/XLSX 处理提供研究型地图
- 服务内部复盘与后续工具链设计

## 范围

包含：

- 面向文档格式的技能指南与配套脚本
- 示例与参考材料
- 对比分析报告（见 `report.md`）

不包含：

- 可直接产品化的端到端系统
- 完整的公开发布清单（见 **许可与 IP 提示**）

## 仓库结构

- `report.md` — 技术对比报告（ChatGPT vs Claude 文档技能）
- `require.md` / `require.zh.md` — 依赖清单与用途说明
- `ChatGPT-document-skills/` — ChatGPT 侧技能说明与示例
- `claude-code-document-skills/` — Claude Code 技能指南、OOXML 参考与脚本
- `share/` — 共享脚本（含幻灯片相关工具）
- `OpenAI_private_Resources/` — **内部/私有资源（需审查）**

## 关键结论（摘要）

- ChatGPT 依赖专有的一体化工具链与内部渲染/公式引擎。
- Claude 基于开源工具与显式验证步骤，强调透明与可审计性。
- 两者体现了“自动化速度（封闭）”与“可扩展/可检查（开放）”的取舍。
- Claude 在 PPTX 场景引入 HTML → PPTX 路径与模板化工具链。

详见 `report.md`。
